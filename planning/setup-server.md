# Issue: Setup Minimal Application Server

## Ringkasan
Lakukan setup server aplikasi minimum agar project dapat dijalankan kembali dengan entry point `app.js` dan memiliki satu endpoint `GET` sederhana yang menampilkan `hello world`.

## Tujuan
- Menyediakan fondasi server yang sangat minimal untuk memastikan aplikasi bisa berjalan.
- Menyediakan satu endpoint awal untuk verifikasi bahwa server aktif.
- Menjaga implementasi tetap sederhana dan tidak melewati scope instruksi.

## Scope Pekerjaan
### 1. Setup Server
- Siapkan server aplikasi dengan entry point `app.js`.
- Pastikan server dapat dijalankan dalam konteks project yang sudah ada.
- Gunakan pendekatan yang sederhana, jelas, dan mudah dipahami.

### 2. Setup Endpoint Minimum
- Tambahkan satu endpoint `GET` pada route.
- Endpoint tersebut hanya perlu mengembalikan response sederhana berupa `hello world`.
- Endpoint ini digunakan hanya untuk memastikan server berhasil berjalan.

## Hasil Akhir yang Diharapkan
- Aplikasi server dapat dijalankan tanpa menambahkan fitur yang tidak diminta.
- Tersedia satu endpoint `GET` yang menampilkan `hello world`.
- Struktur implementasi tetap ringan dan fokus pada setup server minimum.

## Batasan Scope
- Kerjakan hanya sesuai instruksi ini.
- Jangan menambahkan autentikasi, middleware tambahan, CRUD, business logic, atau integrasi fitur lain.
- Jangan memperluas arsitektur di luar kebutuhan setup server minimum.

## Catatan
- Dokumen ini bersifat high level.
- Detail teknis implementasi akan dikerjakan pada tahap eksekusi.
- Prioritaskan hasil yang bersih, sederhana, dan sesuai batas scope.
